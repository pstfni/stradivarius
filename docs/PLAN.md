# Plan: stradivarius TUI app

## Context
A new standalone repo `stradivarius` — a keyboard-driven TUI for browsing Spotify playlist data saved by `chopin`, selecting tracks into a buffer, and creating new playlists. Chopin's existing client/manager API is used for data fetching and playlist creation; stradivarius adds the interactive layer on top.

## 2. Data Layer (`src/stradivarius/data/`)

### `fetch.py`
- `library(output: Path)` — iterates all user playlists via `chopin.client.endpoints.get_user_playlists()`, then `get_playlist_tracks()` for each, flattens into a single polars DataFrame, writes `~/.stradivarius/library.parquet`.
- Flat row schema: `playlist_name | track_name | artist | album | release_year | added_at | track_uri`
- `release_year` derived from `AlbumData.release_date`.
- `artist` = first element of `TrackData.artists`.

### `schema.py`artists
- Typed dataclass `TrackRow` matching the flat schema — used for type-safe construction before writing Parquet.

---

## 3. CLI (`src/stradivarius/cli.py`)

One commands via `clik`:
- `stradivarius` (default) — launches the TUI (loads Parquet).
And a `sync` flag specify to reload the user library, before loading the app:
- `stradivarius --sync` — runs `sync_library()`, prints progress.

## 4. TUI Layer (`src/stradivarius/tui/`)

### Framework & Theme
- `textual` App with Catppuccin Mocha palette via `catppuccin[textual]`.
- CSS in `stradivarius.tcss`; colors referenced as CSS variables from catppuccin.

### Screens

**`screens/library.py`** — main screen (always visible):
- `TrackTable` widget: textual `DataTable` showing all columns.
- `StatusBar` widget (footer): shows buffer count + shortcut hints.
- `FilterBar` widget (conditionally shown above footer): single-line input for regex filter on the focused column.

**`screens/compose.py`** — modal overlay triggered when creating playlist:
- Name input + description input.
- On confirm: calls `chopin.managers.playlist.create_playlist(name, description)` then `fill(uri, buffer_tracks)`.

### Interactions (keyboard-only)

| Key | Action |
|-----|--------|
| `↑` / `↓` | Navigate rows |
| `←` / `→` | Navigate columns |
| Printable char | Open FilterBar for focused column, pre-fill typed char |
| `Enter` (in FilterBar) | Apply regex filter |
| `Escape` | Clear active filter / close modal |
| `s` | Toggle sort asc/desc on focused column |
| `Space` | Add focused track to buffer |
| `b` | Toggle buffer sidebar (list of buffered tracks) |
| `d` | Remove focused item from buffer (when sidebar focused) |
| `p` | Open compose modal (if buffer non-empty) |
| `q` | Quit |

### State
- `app.df: pl.DataFrame` — full loaded dataset.
- `app.filtered_df: pl.DataFrame` — view after filters applied; `DataTable` rows rebuild on change.
- `app.active_filters: dict[col, regex_str]` — one filter per column, composable.
- `app.buffer: list[TrackRow]` — selected tracks for new playlist.

## 5. Verification

1. `uv run stradivarius sync` — confirm `~/.stradivarius/library.parquet` created with correct schema.
2. `uv run stradivarius` — TUI opens, table populated.
3. Navigate columns/rows with arrows, type a char to confirm filter bar appears and filters rows.
4. Press `s` to confirm sort toggles.
5. `Space` adds to buffer; `p` opens compose modal; confirm playlist appears in Spotify.
